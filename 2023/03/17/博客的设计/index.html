博客的设计

2023/3/17 markdown html 博客搭建，主题

前言

之前为了学习人工智能方便做笔记，而云服务器对于抠搜党来说还是不怎么划算，于是比较简单的通过hexo+github做出了简单的搭建。由于时间和各种因素并没有去美化和完善他，已经过了半年了，想乘着博客的搭建复习以下`markdown`和`html`一些知识。

主题更换

emmm....本来想自己搭建的，但是一环扣一环，我先选择现成的主题来使用，在Themes|Hexo中找到喜欢的主题打开github根据教程clone主题，或者将主题下载到本地hexo主题文件夹中进行使用。

个性化制作

在hexo的`config`中，site处更改作者名和博客标题，头像设置在avatar处添加 >(**空一格**）(图片相对路径)

背景则是在background处如头像设置一致。
description修改类似于个性签名，为了格调更高，加上我的github链接（虽然没啥东西),在friendLinks中添加即可。
想更高级点用icon来替换链接，
通过脚本在主页底部显示访客量，将脚本添加在fooder.ejs中。
文章查询

  • 将config7中的search改为true会实现文档标题查询。

文章置顶

  • 在 Front-Matter 里设置 pinned 作为置顶参数，越大越靠前，默认为 0

js特效

无所谓，我会ctrlc+v. 通过js添加流星背景，鼠标点击特性、鼠标指针特效。最后记住在layout.ejs的中添加相应的js。

问题

1.发现markdown删除线语法前后加`~~`没用，所以最后还是选择用前端的方法加``. 2.在部署完成后，使用 >hexo clean hexo g >hexo serve //用于启动本地端口

发现在本地服务器local上正常运行，但在github域名上没有改变，我觉得应该是没有更新到github上，查询发现应该使用

hexo g
hexo d

部署成功，不过加载挺慢的，后续想增加bgm和live2d。
3.发现笔记全篇放出在主页没有缩略，发现是要在文档中加上<!-- more -->
4.忘记了hexo创建的方法了，应该使用hexo new "title"
